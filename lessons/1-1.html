Урок 1.1: Обзор CSS

Теория

История и эволюция CSS

Ключевые этапы развития CSS

  • 1994: Хокон Виум Ли предложил концепцию CSS
  • 1996: CSS1 становится рекомендацией W3C
  • 1998: CSS2 добавляет позиционирование и z-index
  • 2011: CSS3 вводит модульный подход
  • Сегодня: CSS4 и современные возможности

Почему CSS важен?

Разделение содержания и представления

CSS позволяет отделить визуальное оформление от HTML-структуры, делая код более чистым и поддерживаемым.

Консистентность дизайна

Единые стили можно применять ко множеству страниц, обеспечивая единообразие оформления.

Производительность

Внешние CSS-файлы кэшируются браузером, ускоряя загрузку сайта.

Адаптивность

CSS позволяет создавать отзывчивые дизайны для разных устройств и экранов.

Анатомия CSS-правила

.button {
    color: blue;
    background-color: white;
    padding: 10px 20px;
}
  • Селектор (.button) - определяет, к каким элементам применяются стили
  • Свойство (color) - указывает, что именно мы хотим изменить
  • Значение (blue) - определяет новое значение свойства
  • Декларация (color: blue) - пара свойство-значение
  • Блок объявлений ({...}) - содержит все декларации

Методы добавления CSS

Внешний CSS-файл

<link rel="stylesheet" href="styles.css">
Лучшая производительность Легкое обслуживание Кэширование браузером

Встроенные стили

<style>
    body { background: #fff; }
</style>
Быстрая реализация Смешивание HTML и CSS Сложное обслуживание

Инлайн стили

<div style="color: blue;">
Высокая специфичность Трудно поддерживать Нет переиспользования

Важно помнить

CSS - это не просто язык стилей, это мощный инструмент для создания современных веб-интерфейсов. Правильное понимание его основ критически важно для разработки качественных веб-приложений.
