Урок 8.3 - Мобильная версия корзины

1. Адаптация таблицы для мобильных устройств

На мобильных устройствах таблицы требуют особого подхода к отображению. Рассмотрим основные техники адаптации:

/* 1. Скрытие заголовков таблицы */
.mobile-table thead {
    display: none;    /* Скрываем заголовки на мобильных */
}

/* 2. Преобразование строк в блоки */
.mobile-table tr {
    display: block;    /* Каждая строка становится блоком */
    margin-bottom: 15px;
    border: 1px solid #ddd;
    border-radius: 8px;
}

/* 3. Преобразование ячеек в grid */
.mobile-table td {
    display: grid;
    grid-template-columns: 120px 1fr;  /* Метка + значение */
    padding: 8px;
}

Важные моменты при адаптации таблиц:

  • Используйте атрибут data-label для меток в мобильной версии
  • Обеспечьте достаточные отступы для удобства нажатия
  • Добавьте визуальное разделение между элементами

2. Адаптивные кнопки действий

На мобильных устройствах кнопки должны соответствовать рекомендациям по размерам касания:

/* Базовые стили для мобильных кнопок */
.mobile-button {
    min-height: 44px;      /* Минимальная высота по рекомендациям */
    padding: 12px 24px;    /* Достаточные отступы */
    width: 100%;           /* Кнопка на всю ширину */
    font-size: 16px;       /* Читаемый размер шрифта */
    margin-bottom: 10px;   /* Отступ между кнопками */
}

/* Состояния кнопок */
.mobile-button:active {
    transform: scale(0.98);  /* Эффект нажатия */
}

/* Улучшение доступности */
.mobile-button:focus {
    outline: 2px solid #466A85;
    outline-offset: 2px;
}

3. Мобильные формы

Формы на мобильных устройствах должны быть удобны для заполнения и соответствовать следующим принципам:

/* Оптимизация форм для мобильных */
.mobile-form-input {
    height: 44px;          /* Оптимальная высота для тач-интерфейса */
    font-size: 16px;       /* Предотвращает автоматическое масштабирование на iOS */
    padding: 12px;         /* Удобные отступы */
}

/* Группировка элементов формы */
.mobile-form-group {
    margin-bottom: 20px;   /* Отступ между группами */
}

/* Стилизация меток */
.mobile-form-label {
    font-size: 14px;       /* Компактный размер */
    margin-bottom: 8px;    /* Отступ до поля ввода */
}

4. Отступы и поля

Правильное использование отступов (margin) и полей (padding) критично для мобильного интерфейса. Рассмотрим основные принципы:

/* Базовые правила отступов для мобильных */
.mobile-element {
    padding: 16px;         /* Минимальный отступ для тач-интерфейса */
    margin-bottom: 24px;   /* Отступ между элементами */
}

/* Предотвращение схлопывания отступов */
.mobile-container {
    padding: 1px;          /* Создаёт новый контекст форматирования */
    margin: -1px;          /* Компенсация padding */
}

/* Безопасная зона для устройств с вырезами */
.safe-area {
    padding: env(safe-area-inset-top) env(safe-area-inset-right) 
             env(safe-area-inset-bottom) env(safe-area-inset-left);
}

Рекомендации по использованию отступов:

  • Внешние отступы (margin): Используйте для создания пространства между элементами
  • Внутренние отступы (padding): Применяйте для создания пространства внутри элемента
  • Минимальные размеры: Для тач-интерфейсов используйте отступы не менее 16px
  • Консистентность: Поддерживайте единую систему отступов во всём интерфейсе
