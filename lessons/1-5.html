Урок 1.5: Каскад CSS

Теория

Что такое каскад в CSS?

Каскад - это фундаментальный механизм CSS, определяющий, как комбинируются и применяются стили из разных источников. Это "C" в "CSS" (Cascading Style Sheets).

Источники стилей (по возрастанию приоритета)

1. Стили браузера

Базовые стили, применяемые браузером по умолчанию

/* Chrome default styles */
h1 { display: block; font-size: 2em; margin: 0.67em 0; }
2. Пользовательские стили

Стили, установленные пользователем в настройках браузера

3. Авторские стили

Стили, определенные разработчиком сайта

/* Внешний CSS файл */
h1 { color: blue; }
4. Важные авторские стили

Стили с !important

h1 { color: red !important; }

Правила каскада

1. Специфичность

Более специфичные селекторы имеют приоритет

/* Специфичность: 1 */
p { color: blue; }

/* Специфичность: 10 */
.text { color: red; }  /* Победит */

/* Специфичность: 100 */
#unique { color: green; }  /* Победит над всеми */

2. Порядок объявления

При равной специфичности побеждает последнее объявление

.text { color: blue; }
.text { color: red; }  /* Этот стиль применится */

3. Наследование

Некоторые свойства наследуются дочерними элементами

body {
    font-family: Arial;  /* Наследуется */
    margin: 0;          /* Не наследуется */
}

Наследуемые и ненаследуемые свойства

Наследуемые свойства

  • color
  • font-family
  • font-size
  • font-weight
  • line-height
  • text-align
  • visibility

Ненаследуемые свойства

  • margin
  • padding
  • border
  • background
  • width
  • height
  • position

Управление наследованием

Ключевые слова наследования

/* Принудительное наследование */
.child { color: inherit; }

/* Начальное значение свойства */
.reset { color: initial; }

/* Значение по умолчанию браузера */
.default { color: unset; }

/* Принудительное наследование или initial */
.revert { color: revert; }

Лучшие практики

Избегайте !important

/* Плохо */
.element { color: red !important; }

/* Хорошо */
.specific-element { color: red; }

Используйте специфичность разумно

/* Слишком специфично */
.header .nav ul li a { color: blue; }

/* Лучше */
.nav-link { color: blue; }

Организуйте CSS правильно

Располагайте стили в порядке возрастания специфичности:

  1. Сброс стилей
  2. Элементы
  3. Классы
  4. Модификаторы

Важно помнить

  • Каскад - это не баг, а фича CSS
  • Понимание каскада помогает писать более предсказуемый код
  • Избегайте излишней специфичности
  • Используйте наследование для уменьшения дублирования кода

Практика

Задание

Создайте страницу с несколькими элементами и поэкспериментируйте с каскадом CSS. Попробуйте применить стили из разных источников и посмотрите, какие из них имеют приоритет.
