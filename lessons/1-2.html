Урок 1.2: Разделение задач (SoC)

Теория

Что такое Разделение задач (SoC)?

Разделение задач - это принцип проектирования, согласно которому различные аспекты программы должны быть максимально независимы друг от друга. В веб-разработке это означает разделение:

Структура (HTML)

Отвечает за организацию и семантику контента

<article>
    <h1>Заголовок</h1>
    <p>Содержание</p>
</article>

Представление (CSS)

Определяет внешний вид и стилизацию

article {
    max-width: 800px;
    margin: 0 auto;
    padding: 20px;
}

Поведение (JavaScript)

Обеспечивает интерактивность и функциональность

article.addEventListener('click', 
    () => { /* действие */ }
);

Преимущества разделения задач

Улучшенная поддерживаемость

Изменения в одном аспекте не влияют на другие. Можно обновить дизайн без изменения HTML.

Повторное использование

Один CSS-файл может применяться к множеству HTML-страниц.

Командная работа

Разные специалисты могут работать над разными аспектами параллельно.

Производительность

Кэширование CSS и JavaScript улучшает скорость загрузки.

Распространенные анти-паттерны

❌ Смешивание стилей с HTML

<p style="color: blue; font-size: 16px;">
    Текст с инлайн-стилями
</p>

✅ Решение: Вынести стили в CSS-файл

❌ Стилизация через HTML-атрибуты

<font color="red" size="4">
    Устаревший подход
</font>

✅ Решение: Использовать современные CSS-свойства

❌ Презентационные классы

<div class="blue bold size-14">
    Негибкий подход
</div>

✅ Решение: Использовать семантические классы

Лучшие практики

Семантическая разметка

❌ Плохо:
<div class="header">
    <div class="title">Заголовок</div>
</div>
✅ Хорошо:
<header>
    <h1>Заголовок</h1>
</header>

Организация CSS

/* Компонентный подход */
.article { }
.article__header { }
.article__content { }
.article--featured { }

Используйте методологии типа BEM для структурирования CSS

Прогрессивное улучшение

Начинайте с базовой функциональности и добавляйте улучшения постепенно:

  1. Семантический HTML для всех пользователей
  2. Базовые стили для старых браузеров
  3. Продвинутые стили с современными свойствами
  4. JavaScript для дополнительной функциональности

Важно помнить

Разделение задач - это не просто техническое решение, а философия разработки, которая помогает создавать более качественные, поддерживаемые и масштабируемые веб-приложения.

Практика

Задание

Создайте простую веб-страницу с карточкой товара, применяя принцип разделения задач. Разделите структуру (HTML) и стили (CSS).
