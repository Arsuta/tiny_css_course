Урок 1.7: Способы написания CSS

Теория

Сокращенная запись (Shorthand)

CSS предоставляет возможность использовать сокращенную запись для некоторых свойств, что делает код более компактным и читаемым.

Margin и Padding

/* Полная запись */
margin-top: 10px;
margin-right: 20px;
margin-bottom: 10px;
margin-left: 20px;

/* Сокращенная запись */
margin: 10px 20px;

Border

/* Полная запись */
border-width: 1px;
border-style: solid;
border-color: black;

/* Сокращенная запись */
border: 1px solid black;

Background

/* Полная запись */
background-color: #fff;
background-image: url('image.jpg');
background-repeat: no-repeat;
background-position: center;

/* Сокращенная запись */
background: #fff url('image.jpg') no-repeat center;

Форматирование кода

Рекомендуемый стиль

/* Хорошее форматирование */
.button {
    display: inline-block;
    padding: 10px 20px;
    background-color: #007bff;
    color: white;
    border-radius: 4px;
}

/* Группировка связанных свойств */
.button {
    /* Позиционирование */
    position: relative;
    z-index: 1;

    /* Размеры */
    width: 200px;
    height: 50px;

    /* Внешний вид */
    background: #007bff;
    border-radius: 4px;
    color: white;
}

Рекомендации по форматированию

  • Используйте отступы для вложенных правил
  • Группируйте связанные свойства
  • Добавляйте пробел после двоеточия
  • Ставьте каждое свойство на новой строке
  • Используйте комментарии для описания сложных участков

Методологии написания CSS

Существует несколько популярных подходов к организации CSS-кода, каждый со своими преимуществами и особенностями.

BEM (Block Element Modifier)

Заголовок карточки

Содержимое карточки 
.card { }
.card__title { }
.card__content { }
.card--featured { }

BEM обеспечивает четкую и понятную структуру именования классов:

  • Блок: самостоятельная сущность
  • Элемент: часть блока (__)
  • Модификатор: вариация блока/элемента (--)

OOCSS (Object-Oriented CSS)

Светлая тема

Пример разделения структуры и оформления

Темная тема

Тот же компонент, другой скин

/* Структура (независима от оформления) */
.structure {
    padding: 1rem;
    border-radius: 8px;
    margin: 1rem 0;
}

/* Скины (независимы от структуры) */
.skin-light {
    background: white;
    color: #333;
    border: 1px solid #ddd;
}

.skin-dark {
    background: #333;
    color: white;
    border: 1px solid #666;
}

OOCSS разделяет структуру и оформление:

  • Структура: размеры, отступы, позиционирование
  • Скин: цвета, границы, тени
  • Многократное использование стилей

SMACSS (Scalable and Modular Architecture for CSS)

Пример макета (Layout)

Активная ссылка Обычная ссылка Еще ссылка 
/* Базовые стили */
a { color: #666; }

/* Стили макета */
.layout-header { 
    width: 100%;
    padding: 1rem;
}

/* Модули */
.nav-item {
    padding: 0.5rem 1rem;
}

/* Состояния */
.state-active {
    color: #3498db;
    font-weight: bold;
}

SMACSS разделяет стили на категории:

  • Base: базовые стили элементов
  • Layout: макет и сетка
  • Module: независимые компоненты
  • State: состояния элементов
  • Theme: темы оформления

CSS Modules

Сгенерированный класс: button_primary_abc123

/* Button.module.css */
.button {
    background: var(--primary-color);
    color: white;
    padding: 0.5rem 1rem;
    border-radius: 4px;
}

/* React Component */
import styles from './Button.module.css';

function Button() {
    return ;
}

CSS Modules автоматически создают уникальные имена классов:

  • Локальная область видимости стилей
  • Нет конфликтов имен
  • Лучшая интеграция с компонентами

CSS-in-JS

Hover State
const Button = styled.button`
    background: ${props => props.primary 
        ? 'var(--primary-color)' 
        : 'gray'};
    color: white;
    padding: 0.5rem 1rem;
    border-radius: 4px;
    transition: transform 0.2s;

    &:hover {
        transform: translateY(-2px);
    }
`;

CSS-in-JS предлагает:

  • Динамические стили на основе props
  • Изоляцию стилей на уровне компонентов
  • Автоматическое удаление неиспользуемых стилей
  • Поддержку тем и динамических значений

CSS-препроцессоры

SASS/SCSS

// Переменные
$primary-color: #007bff;

// Вложенность
.card {
    &__title {
        color: $primary-color;
    }
    
    &--featured {
        background: lighten($primary-color, 40%);
    }
}

Less

// Миксины
.button-styles() {
    padding: 10px;
    border-radius: 4px;
}

.button {
    .button-styles();
    background: @primary-color;
}

Современные подходы

CSS Modules

/* button.module.css */
.button {
    background: var(--primary-color);
}

/* Использование */
import styles from './button.module.css';

CSS-in-JS

const Button = styled.button`
    background: ${props => props.primary ? 'blue' : 'gray'};
    padding: 10px 20px;
    border-radius: 4px;
`;

Рекомендации по выбору подхода

  • Для небольших проектов: простая организация по компонентам
  • Для средних проектов: BEM + препроцессор
  • Для крупных проектов: CSS Modules или CSS-in-JS
  • Для команд: согласованный подход важнее конкретной методологии
