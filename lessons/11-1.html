Урок 11.1 - Работа с CSS архитектурой

OOCSS (Object-Oriented CSS)

OOCSS (Object-Oriented CSS) - это методология CSS, разработанная Николь Салливан. Она основана на принципах объектно-ориентированного программирования и направлена на создание многократно используемых компонентов интерфейса.

Основные принципы:

  • Разделение структуры и оформления

    Базовая структура (размеры, отступы) отделяется от визуального оформления (цвета, границы). Это позволяет создавать разные темы, не меняя структуру.

  • Разделение контейнера и содержимого

    Стили не должны зависеть от расположения элемента в DOM. Компонент должен выглядеть одинаково независимо от его местоположения.

  • Повторное использование стилей

    Создание абстрактных классов, которые можно комбинировать для получения нужного результата.

Преимущества OOCSS:

  • Меньший размер CSS - за счет переиспользования стилей
  • Лучшая производительность - меньше дублирования кода
  • Проще поддерживать - изменения вносятся в одном месте
  • Масштабируемость - легко добавлять новые компоненты
/* OOCSS пример */
.btn {  /* Структура */
    padding: 10px 20px;
    border-radius: 4px;
    border: none;
}

.btn-primary {  /* Оформление */
    background: #466A85;
    color: white;
}

SMACSS (Scalable and Modular Architecture for CSS)

SMACSS - это методология, разработанная Джонатаном Снуком, которая предлагает более гибкий подход к организации CSS. Основная идея заключается в категоризации CSS-правил и создании руководств для каждой категории.

Категории SMACSS:

  • Base

    Стили по умолчанию для HTML-элементов (h1-h6, p, a и т.д.). Используются селекторы элементов без классов.

  • Layout

    Стили для основных структурных компонентов (header, footer, sidebar). Префикс "l-" или "layout-".

  • Module

    Стили для повторно используемых компонентов интерфейса (кнопки, формы, карточки).

  • State

    Стили для различных состояний элементов (активный, скрытый). Префикс "is-" или "has-".

  • Theme

    Переопределение базовых стилей для создания вариаций внешнего вида. Префикс "theme-".

Правила именования в SMACSS:

  • Layout: .l-header, .l-sidebar, .l-grid
  • State: .is-active, .is-hidden, .has-error
  • Theme: .theme-dark, .theme-light
  • Module: .card, .btn, .nav
/* SMACSS пример */
.l-header { /* Layout */
    width: 100%;
}

.is-active { /* State */
    background: #e6f3ff;
}

.theme-dark { /* Theme */
    background: #333;
    color: #fff;
}

BEM (Block Element Modifier)

BEM - это методология именования CSS-классов, разработанная в Яндексе. Основная идея заключается в разделении интерфейса на независимые блоки и создании понятной и строгой системы именования классов.

Структура BEM:

  • Блок (Block)

    Функционально независимый компонент, который можно повторно использовать. Например: .header, .menu, .button

  • Элемент (Element)

    Составная часть блока, которая не может использоваться отдельно. Обозначается двойным подчеркиванием: .block__element

  • Модификатор (Modifier)

    Определяет внешний вид, состояние или поведение блока/элемента. Обозначается двойным дефисом: .block--modifier или .block__element--modifier

Примеры именования в BEM:

  • Блок: .menu
  • Элемент: .menu__item
  • Модификатор блока: .menu--vertical
  • Модификатор элемента: .menu__item--active

Заголовок карточки

Содержимое карточки 
/* BEM пример */
.card {  /* Блок */
    padding: 20px;
}

.card__title {  /* Элемент */
    font-size: 20px;
}

.card--featured {  /* Модификатор */
    border: 2px solid #466A85;
}

Другие методологии CSS

Помимо OOCSS, SMACSS и BEM существуют и другие подходы к организации CSS:

Atomic CSS

Подход, при котором каждый класс отвечает за одно CSS-свойство. Например:

  • .m-0 - margin: 0
  • .p-1 - padding: 1rem
  • .flex - display: flex
  • .text-center - text-align: center

ITCSS (Inverted Triangle CSS)

Методология, организующая CSS-файлы по уровням специфичности:

  • Settings - переменные и конфигурация
  • Tools - миксины и функции
  • Generic - сброс стилей
  • Elements - стили элементов
  • Objects - паттерны макета
  • Components - компоненты UI
  • Utilities - утилиты и перезаписи

CSS Modules

Технический подход, при котором стили автоматически получают уникальные имена классов:

/* CSS Module пример */
.button {
    background: #466A85;
}

/* Компилируется в */
.button_abc123 {
    background: #466A85;
}

Utility-First CSS

Подход, популяризированный Tailwind CSS, где весь дизайн строится из утилитарных классов:


<div class="flex items-center p-4 bg-white rounded-lg shadow-md">
    <h2 class="text-xl font-bold text-gray-800">Заголовок</h2>
</div>
